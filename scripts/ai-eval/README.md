# AI Codegen Evals for @foblex/flow

Measures whether LLMs/AI agents can produce **working** Foblex Flow code from our AI-facing
docs (`AI.md`, `llms-full.txt`, the `AGENTS.md` block installed by `ng add`) — and catches
regressions when the API or the docs change.

## Approach

Use Angular's [web-codegen-scorer](https://github.com/angular/web-codegen-scorer) with a
custom f-flow environment: each prompt below is generated by the model under test into a
fresh Angular app with `@foblex/flow` installed, then scored on build success, runtime
errors, and the acceptance checks.

Run each prompt in two configurations and compare pass rates:

1. **Baseline** — no extra context (what a model does from training data alone).
2. **With docs** — the app contains the `AGENTS.md` block from `ng add @foblex/flow`
   (agents read `node_modules/@foblex/flow/AI.md`).

The delta between the two is the measured value of the docs; a drop after a release means
the docs no longer match the API.

## Acceptance checks (all prompts)

- `ng build` succeeds.
- No `FFxxxx` warnings/errors in the dev console after first render (see
  https://flow.foblex.com/docs/errors).
- `(fFullRendered)` fires and `flow.getState()` shows every declared connection resolved.
- No React Flow API hallucinations (`[nodes]`, `[edges]`, `addEdge`, `<Handle>`).

## Prompts

1. Render a flow with two draggable nodes connected by an edge. The connection must
   survive dragging both nodes.
2. Build a flow where the user can drag a new connection from one node to another; persist
   it in component state so it survives change detection.
3. Add rectangle multi-select: dragging on the empty canvas selects the covered nodes.
4. Make the mouse wheel zoom the canvas, limited between 0.5x and 2x.
5. Switch the editor to a Miro-like control scheme: scroll pans, Ctrl/Cmd+scroll zooms,
   drag on empty canvas selects.
6. Add a node palette outside the canvas; dragging an item onto the canvas creates a node
   at the drop position.
7. Group three nodes; the group must auto-size to fit its children and move them together.
8. Allow reassigning an existing connection's target to another node and reflect it in
   state.
9. Add resize handles to a node and keep its size in component state.
10. Render 500 nodes progressively without blocking the main thread.

## Running

```bash
npx web-codegen-scorer eval --env=./scripts/ai-eval/environment  # once an environment config is added
```

The environment config (system prompt, project scaffold with @foblex/flow installed, and
per-prompt checks) still needs to be authored against the scorer's current config format —
see the scorer repo's `environments/` examples. Track results per release next to this
file.
