# Tournament Bracket

Tournament-bracket reference app built with Angular and Foblex Flow.

## User-Facing Functionality

- Browse a full elimination bracket on a pannable and zoomable canvas.
- Switch between standard, compact, and mirrored layout strategies.
- Toggle upper, lower, and final brackets independently.
- Inspect live counters for played, live, and upcoming matches.
- Select a match to highlight its path through the bracket.
- Select a team to highlight its journey and show team stats.
- Open a detail panel with match timing, status, score, and team summary.
- Use minimap, fit-to-screen, reset zoom, and theme toggle controls.

## Foblex Flow Building Blocks Used

- `f-flow` as the root interaction surface.
- `f-canvas` with `fZoom` for viewport navigation.
- `fNode` for match cards placed by a custom layout engine.
- `fConnection` with fixed segmented routing for bracket links.
- `fConnectionMarkerArrow` for directional relation markers.
- `fMinimap` for overview navigation on large bracket graphs.
- `provideFLayout(...)` to plug a domain-specific layout engine into Flow.

## Integration Notes

- The bracket is intentionally domain-specific: match positions are not hand-authored in the template, they are generated by a custom layout engine that lives inside the app.
- Visibility filters update both rendered nodes and rendered connections, so the canvas stays consistent when entire branches are hidden.
- Selection and highlight logic are stored in a local signal-based state service and projected back into node and connection styling.

## Run Locally

From this directory:

```bash
npm run dev
npm run build
npm run lint
```

Or from the workspace root:

```bash
npx nx serve tournament-bracket --configuration development
```
